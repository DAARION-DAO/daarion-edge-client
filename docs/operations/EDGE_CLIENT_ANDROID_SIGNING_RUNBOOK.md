# DAARION Edge Client — Android Signing Runbook

## Current Blocker
The Android APK generated by the CI workflow is currently unsigned (or signed with a debug key). Real Android devices and the Google Play Store reject unsigned or debug-signed APKs for release distributions. To produce an installable release APK, the build process must use a valid release keystore and corresponding signing configuration.

## Required GitHub Secrets
The operator must add the following secrets to the GitHub repository:

- `ANDROID_KEYSTORE_BASE64` — The base64-encoded `daarion-edge-release.keystore` file.
- `ANDROID_KEYSTORE_PASSWORD` — The password for the keystore.
- `ANDROID_KEY_ALIAS` — The alias of the key (e.g., `daarion-edge`).
- `ANDROID_KEY_PASSWORD` — The password for the key.

## Generating the Android Release Keystore Locally
1. Run the following command to generate a new 2048-bit RSA keystore valid for 10000 days:
   ```bash
   keytool -genkeypair \
     -v \
     -keystore daarion-edge-release.keystore \
     -alias daarion-edge \
     -keyalg RSA \
     -keysize 2048 \
     -validity 10000
   ```
2. When prompted, enter a strong password for both the keystore and the key, and fill in the certificate details.

## Encoding the Keystore for GitHub Secrets
1. Base64-encode the generated `.keystore` file and copy it to your clipboard:
   ```bash
   base64 -i daarion-edge-release.keystore | pbcopy
   ```
2. Paste this value into the `ANDROID_KEYSTORE_BASE64` secret in GitHub.

> **⚠️ SECURITY RULES:**
> - DO NOT commit the `daarion-edge-release.keystore` file to the repository.
> - DO NOT commit the decoded or encoded keystore.
> - DO NOT print GitHub Secrets in the CI workflow.

## Triggering the Release Workflow
Once the secrets are added, trigger the GitHub Actions release workflow:
1. Push a new tag starting with `v` (e.g., `v0.2.1`), OR
2. Manually trigger the "DAARION Edge Client — Sovereign Release" workflow from the GitHub Actions tab.

## Verifying the Signed APK with ADB
1. Download the generated `.apk` from the GitHub Release artifacts.
2. Connect your Android device via USB and ensure USB debugging is enabled (or use an emulator).
3. Install the APK:
   ```bash
   adb install -r <signed-apk-file>.apk
   ```
4. Verify the installation by launching the app:
   ```bash
   adb shell monkey -p city.daarion.edge 1
   ```
